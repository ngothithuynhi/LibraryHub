# LibraryHub Contribution Guide

## Branching Strategy

main -> final stable branch

develop -> integration branch

feature/* -> working branch

Examples:

feature/auth
feature/book-crud
feature/borrow-api

## Pull Request Rules

- No push directly to main
- All changes must go through Pull Request
- At least 1 reviewer approval before merge

## Commit Convention

feat: add new feature

fix: fix bug

test: add tests

docs: update documentation

chore: setup/config changes
